# Implementation Plan: Sign In Page

## Goal
Implement a fully functional `/login` page with Supabase email/password auth, Radix UI `Label` components, accessible form UX, and all styles driven by a centralized CSS-var theme — no hardcoded hex values.

## Research Summary
- Project: Next.js 16 App Router + TypeScript + Tailwind CSS v4
- `/login` route exists as a stub (`src/app/login/page.tsx`)
- No Supabase or Radix UI packages installed — must be added
- Theme lives in `src/app/globals.css` via CSS custom properties + `@theme inline` (Tailwind v4 pattern)
- Post-login redirect target: `/` (`src/app/page.tsx`)
- Data flow: form → `supabase.auth.signInWithPassword()` → redirect or show error

### Phase 4: Implement the Sign In Page
**Status**: ⏳ Pending

**Dependencies**: Phases 1, 2, 3

**Files to Modify:**
- `src/app/login/page.tsx` — replace stub with full implementation

**Changes:**

Add `'use client'` directive (client component — needs React state + router).

UI structure:
- `<main>` with `min-h-screen`, centered via flexbox
- Sign-in card: `<div>` with `bg-card`, `text-card-foreground`, `border-border`, rounded corners, padding, shadow
- Heading: `<h1>` "Sign In"
- `<form>` with `onSubmit` handler
- Email field: `<Label>` (Radix) + `<input type="email">` styled with Tailwind (`bg-background`, `border-border`, `text-foreground`, `ring-ring` on focus)
- Password field: `<Label>` (Radix) + `<input type="password">` same styling
- Error message: conditionally rendered `<p role="alert">` with `text-destructive` — shown on auth failure
- Submit button: `<button type="submit">` with `bg-primary`, `text-primary-foreground`, disabled + loading spinner when `isLoading` is true

State:
- `email: string`
- `password: string`
- `error: string | null`
- `isLoading: boolean`

Form submit handler:
1. Set `isLoading = true`, clear `error`
2. Call `createClient().auth.signInWithPassword({ email, password })`
3. On success: `router.push('/')` (using `useRouter` from `next/navigation`)
4. On error: set `error` to the Supabase error message, set `isLoading = false`

Accessibility:
- All `<input>` elements have `id` matching `<Label htmlFor>`
- Submit button has `aria-busy={isLoading}`
- Error `<p>` has `role="alert"` and `aria-live="polite"`
- Tab order is natural (email → password → button)
- `Enter` key submits (native form behavior)

- [ ] Submitting with wrong credentials shows error message
- [ ] Submitting with valid credentials redirects to `/`
- [ ] Button shows disabled state while submitting
- [ ] Keyboard-only navigation works (Tab + Enter)
- [ ] Dark mode renders correctly (card, inputs, button all use theme tokens)
- [ ] No hardcoded hex values in the component

**Success Criteria:** Fully functional, accessible, themed sign-in page.

### Phase 5: Add Middleware for Session Refresh (Optional)
**Status**: ⏳ Pending

**Dependencies**: Phases 1, 3

**Files to Create:**
- `middleware.ts` (project root)

**Changes:**
Use `@supabase/ssr` `createServerClient` in middleware to refresh the session cookie on each request — prevents silent session expiry.

```ts
// middleware.ts
import { createServerClient } from '@supabase/ssr'
import { NextResponse, type NextRequest } from 'next/server'

export async function middleware(request: NextRequest) {
  let supabaseResponse = NextResponse.next({ request })
  const supabase = createServerClient(
    process.env.NEXT_PUBLIC_SUPABASE_URL!,
    process.env.NEXT_PUBLIC_SUPABASE_ANON_KEY!,
    {
      cookies: {
        getAll() { return request.cookies.getAll() },
        setAll(cookiesToSet) {
          cookiesToSet.forEach(({ name, value, options }) =>
            supabaseResponse.cookies.set(name, value, options)
          )
        },
      },
    }
  )
  await supabase.auth.getUser()
  return supabaseResponse
}

export const config = {
  matcher: ['/((?!_next/static|_next/image|favicon.ico).*)'],
}
```

**Verification:**
- [ ] `npm run dev` — no middleware errors in console
- [ ] `npm run build` — compiles without errors

**Success Criteria:** Session cookies refreshed automatically on each request.

**Estimated Complexity**: Low

## Risks and Considerations
- **Env vars**: The page will render but Supabase calls will fail without a real `.env.local`. Developers must configure this before testing auth.
- **Tailwind v4 token naming**: `@theme inline` requires `--color-*` prefix to generate Tailwind utilities. Wrong naming won't produce usable classes.
- **`createBrowserClient` vs `createClient`**: Only use the browser client in `'use client'` components. Server components need `createServerClient`.
